# TypeScript 규칙 시드

최소 시드입니다. 프로젝트가 실제로 TypeScript를 사용할 때만 이 파일을 확장합니다.

- 기존 패키지 매니저와 lockfile을 존중합니다.
- 타입 안정성을 유지합니다. 광범위한 `any`나 포괄적 억제를 피합니다.
- 프로젝트에서 설정됐다면 `prettier`로 포맷합니다.
- 프로젝트에서 설정됐다면 `eslint`로 린트합니다.
- 프로젝트에서 설정됐다면 동작 변경을 주장하기 전에 가장 좁은 범위의 `tsc` 또는 테스트 명령을 실행합니다.
- 새 도구를 추가하기 전에 기존 프로젝트 관례를 먼저 따릅니다.
