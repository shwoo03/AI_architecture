# Goal Prompt Rules

## 한 문장 정의

`goal 프롬프트`는 Codex CLI의 `/goal` 기능에 넣을 짧은 실행 프롬프트를 뜻합니다.

## 무엇을 하는가

사용자가 `goal 프롬프트`를 달라고 하면 에이전트는 일반 설명문이나 장문 계획서가 아니라 `/goal`에 바로 붙여 넣을 수 있는 프롬프트를 작성합니다. 프롬프트는 4000자 이하로 유지합니다.

## 왜 필요한가

`/goal` 프롬프트가 너무 길면 CLI 입력으로 다루기 어렵고, 목표·제약·검증 기준이 한 덩어리로 섞여 다음 세션이 오히려 불안정해집니다. 긴 내용은 문서로 분리하고 `/goal`은 그 문서를 가리키는 짧은 진입점이어야 합니다.

## 어떻게 동작하는가

에이전트는 다음 규칙을 따릅니다.

- `goal 프롬프트` 요청은 Codex CLI `/goal` 요청으로 해석합니다.
- 출력하는 `/goal` 프롬프트는 4000자 이하로 씁니다.
- 4000자를 넘을 내용이면 먼저 별도 문서에 상세 범위와 수용 기준을 정리합니다.
- 그 다음 `/goal` 프롬프트는 해당 문서 경로를 명시하고, 실행 목표와 핵심 제약만 짧게 담습니다.
- 작성 후 필요하면 `python3 scripts/goal-prompt-check.py --text "<prompt>"` 또는 stdin으로 길이를 확인합니다.

## 긴 프롬프트 처리

긴 프롬프트가 필요할 때는 다음 형태를 사용합니다.

```text
/goal <짧은 목표>. 상세 범위와 수용 기준은 docs/goal-prompts/<파일명>.md를 기준으로 따르라. 문서 범위를 벗어나지 말고, 검증 후 closeout까지 수행하라.
```

## 결과

완성된 goal 프롬프트는 다음 조건을 만족해야 합니다.

- Codex CLI `/goal`에 넣는 용도임이 명확합니다.
- 4000자 이하입니다.
- 긴 세부사항은 문서 경로로 분리되어 있습니다.
- 구현 범위, 금지 범위, 검증 기준 중 핵심만 남깁니다.
