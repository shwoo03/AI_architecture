# Reference Wiki

## 한 문장 정의

Reference Wiki는 외부 레퍼런스에서 얻은 원본 자료와 정리된 운영 지식을 분리해 저장하는 로컬 지식 구조입니다.

## 무엇을 하는가

- `raw/`에는 원본 메모, 출처 요약, 분석 노트를 둡니다.
- `wiki/`에는 후보 카드나 제안서로 승격할 수 있는 정리본을 둡니다.
- `wiki/index.md`는 주제, 상태, source pointer를 추적합니다.
- `wiki/log.md`는 지식 정리의 변경 이력을 날짜별로 남깁니다.

## 왜 필요한가

레퍼런스 분석 결과가 채팅이나 임시 메모에만 남으면 다음 세션이 같은 저장소를 다시 훑게 됩니다. 원본과 정리본을 분리하면 출처와 판단을 보존하면서도 후보 카드와 dry-run proposal로 안전하게 승격할 수 있습니다.

## 어떻게 동작하는가

1. 레퍼런스 repo, 문서, sub-agent 분석 결과를 `raw/`에 요약합니다.
2. 반복 가치가 있는 패턴만 `wiki/` 문서로 정리합니다.
3. `wiki/index.md`에 상태와 source pointer를 추가합니다.
4. 후보 카드나 proposal로 승격하면 `wiki/log.md`에 기록합니다.

## 결과는 무엇인가

- 원본 출처와 정리된 지식이 분리됩니다.
- reference adoption 전에 근거를 다시 확인할 수 있습니다.
- 오래된 지식, 중복 지식, 승격된 지식을 추적할 수 있습니다.
