# 예시 프로젝트 프로필

## ⚠ 이 예시들은 “완성된” 프로필입니다

각 예시(`examples/*/PROJECT_PROFILE.md`)는 프로젝트가 이미 진행되어 `active_workflows`, `active_skills`, `active_rules`, `validation_summary`, `pivot_summary`까지 정해진 **성숙 상태** 스냅샷입니다. **신규 프로젝트가 처음부터 이 모든 값을 알고 있을 필요는 없습니다.** Cold start 대화에서는 목표·성공 기준·실패 기준까지만 채우고, “활성 운영 선택” 섹션은 비워 두거나 `TBD`로 둡니다. 에이전트가 프로젝트 진행 중 반복 패턴을 발견하면 그때 제안합니다.

## 한 문장 정의

이 문서는 서로 다른 도메인의 프로젝트 프로필 예시를 제공해 사용자가 “프로필을 어떻게 채우는지” 빠르게 이해하게 합니다.

## 무엇을 하는가

예시는 실제 프로젝트에 그대로 복사하는 정답이 아니라, 목표, 성공 기준, 제약 조건, 활성 워크플로를 어떤 수준으로 작성하는지 보여주는 참고 자료입니다.

## 왜 필요한가

빈 템플릿만 보면 사용자는 어느 정도 구체적으로 써야 하는지 판단하기 어렵습니다. 도메인이 다른 예시를 보면 프로젝트 프로필이 장황한 기획서가 아니라 얇은 운영 계약이라는 점을 이해하기 쉽습니다.

## 어떻게 동작하는가

새 프로젝트를 시작할 때 먼저 예시를 읽어 표현 수준을 파악합니다. 그 다음 현재 프로젝트의 사실만 사용해 새 프로젝트 프로필을 작성합니다. 예시의 도메인, 제약, 성공 기준을 현재 프로젝트에 맞는 사실처럼 복사하지 않습니다.

## 결과는 무엇인가

예시를 사용하면 다음을 얻습니다.

- 프로필 작성 수준을 빠르게 이해합니다.
- 성공 기준과 실패 기준을 구체적으로 쓰는 방식을 볼 수 있습니다.
- 에이전트 부트스트랩 대화에서 few-shot 참고 자료로 사용할 수 있습니다.
- 서로 다른 도메인에서도 같은 프로필 구조를 유지합니다.

## 포함된 예시

| 예시 도메인 | 보여주는 판단 |
| --- | --- |
| CTF 풀이와 익스플로잇 개발 | 짧은 시간 안에 문제 풀이 성공 기준을 잡는 방식 |
| 데이터 파이프라인 | 데이터 품질, 재처리, 검증 기준을 잡는 방식 |
| 기술 문서화 시스템 | 문서 품질과 지식 축적 기준을 잡는 방식 |
| 학습 시스템 | 학습 성과와 사용자 피드백 기준을 잡는 방식 |
| 풀스택 웹 애플리케이션 | 기능 출시와 사용자 경험 기준을 잡는 방식 |

## 기능 추가/수정 판단 기준

새 예시는 다음 조건을 만족할 때 추가합니다.

- 기존 예시와 다른 도메인 판단을 보여줍니다.
- 목표, 성공 기준, 실패 기준, 제약 조건이 구체적입니다.
- 다른 사용자가 참고해도 잘못 복사할 위험이 낮습니다.

기존 예시는 현재 프로젝트에 맞추려고 수정하지 않습니다. 예시는 참조 자료이며, 활성 프로젝트 상태가 아닙니다.
