# 0010-ownership-skeleton-self-classification

## Summary

Ownership-aware upgrade v1 Slice 2.5 proves that the AI_architecture skeleton can classify itself using `config/ownership.yaml`. This slice adds the initial generated ownership lock and self-classification tests, without integrating ownership checks into `verify-skeleton.py` or `upgrade-from-skeleton.py`.

## Assumptions

- Slice 2 classifier library and synthetic tests exist.
- `0007-role-registry-audit` remains active/deferred and untouched.
- The lock file is generated by tooling, not edited by hand.
- ENKI migration remains out of scope until Slice 4 dry-run.

## Out of Scope

- `verify-skeleton.py` ownership check integration.
- `upgrade-from-skeleton.py` ownership-aware application.
- ENKI_WIKI dry-run or migration.
- New ownership categories or schema version changes.

## Implementation Steps

- Add `scripts/ownership-lock.py` with `write` and `check` commands.
- Extend `scripts/lib_ownership.py` with repository path collection, self-classification, lock payload generation, lock write/load, and root-vs-lock comparison helpers.
- Generate `runtime/ownership-classification.lock.json` from the current skeleton.
- Add `tests/test_skeleton_self_classification.py` for no-unknown ownership, expected core path classifications, source==target idempotence, and lock drift absence.
- Register the new tool in `scripts/catalog.yaml`, help tests, and `config/ownership.yaml`.

## Definition of Done

- `scripts/ownership-lock.py write` creates `runtime/ownership-classification.lock.json`.
- `scripts/ownership-lock.py check` exits 0 against the current skeleton.
- Self-classification reports no unknown durable paths.
- Existing validation passes.

## Rollback Plan

- Remove `scripts/ownership-lock.py`, `tests/test_skeleton_self_classification.py`, and `runtime/ownership-classification.lock.json`.
- Revert the related entries in `scripts/lib_ownership.py`, `scripts/catalog.yaml`, `tests/test_validation.py`, `config/ownership.yaml`, and `plans/INDEX.md`.

## Stop Conditions

- If self-classification exposes unknown paths that require broad new policy decisions, stop and report the unknown set.
- If lock generation requires changing `verify-skeleton.py`, stop and move that to Slice 3.
- If generated/cache paths dominate the lock, stop and tighten `skip_patterns` before writing the lock.
