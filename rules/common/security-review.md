# 보안 리뷰 규칙

비밀값, 인증, 권한, 데이터 접근, 네트워크 호출, 생성된 코드 실행, 자동화를 다루는 작업에 이 규칙을 적용합니다.

확인 항목:

- 비밀값이 커밋되거나 로그에 남지 않아야 합니다.
- 쓰기 범위가 명시적이어야 합니다.
- 생성된 변경은 기본적으로 dry-run 상태여야 합니다.
- 파괴적 작업은 명시적 승인을 요구합니다.
- runtime 로그가 자격 증명이나 민감한 개인 정보를 담지 않아야 합니다.
- 에이전트 권한이 스스로를 확장하지 않아야 합니다.

샌드박스, 승인 절차, 자격 증명 경계를 바꾸는 행동은 사람에게 에스컬레이션합니다.
