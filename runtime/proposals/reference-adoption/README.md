# 외부 사례 적용 제안

## 한 문장 정의

이 디렉터리는 외부 후보 카드에서 뽑은 패턴을 실제 스켈레톤 변경으로 옮기기 전에 검토하는 dry-run 제안 공간입니다.

## 무엇을 하는가

후보 카드는 외부 사례 자체를 평가합니다. 이 디렉터리의 제안서는 그 후보를 우리 스켈레톤에 적용한다면 어떤 파일을 어떻게 바꿀지, 무엇을 적용하지 않을지, 어떤 검증을 할지 보여줍니다.

제안서는 실제 반영이 아닙니다. 승인 전에는 운영 문서, 규칙, 스킬, 스크립트를 바꾸지 않습니다.

사용자는 제안서 파일을 직접 수정하지 않습니다. 채팅에서 승인, 거절, 보류 의사를 밝히면 에이전트가 제안서의 상태와 최종 결정 기록, 후보 카드, 활동 로그, 세션 인수인계를 갱신합니다.

제안서에는 두 종류가 있습니다.

- 예시 제안: 후보 카드와 dry-run 제안 흐름이 작동하는지 보여주는 샘플입니다. 특정 프레임워크 도입 계획이 아닙니다.
- 실제 채택 제안: 사용자 승인을 받으면 지정된 파일 변경으로 이어질 수 있는 제안입니다.

예시 제안은 제목과 한 문장 정의에 예시 성격을 명시합니다.

## 왜 필요한가

외부 사례는 좋아 보여도 그대로 가져오면 스켈레톤이 무거워지거나 의존성이 늘 수 있습니다. 제안서는 "좋은 외부 패턴"을 "우리 프로젝트에 맞는 작은 변경"으로 번역하는 중간 단계입니다.

## 어떻게 동작하는가

1. `research/reference-candidates/`의 후보 카드에서 `adopt`, `adapt`, `copy` 결정을 확인합니다.
2. 후보 카드의 `next_action`을 하나의 제안서로 만듭니다.
3. 제안서에는 변경 대상, 변경하지 않을 것, 검증 계획, 중단 조건을 적습니다.
   `copy` 또는 `partial_copy`라면 원본 라이선스, 확인한 revision, 복사할 파일/함수/스니펫, 로컬 전용 조건, 재배포 전 재검토 조건도 적습니다.
4. 사용자가 승인하면 별도 작업으로 실제 파일을 수정합니다.
5. 적용 후 후보 카드와 활동 로그에 반영 결과를 남깁니다.

파일 이름은 다음 형식을 사용합니다.

```text
YYYY-MM-DD-<candidate>-<topic>.md
```

새 제안은 `_template.md`를 복사해 작성합니다. 제안서 형식은 다음 명령으로 확인합니다.

```powershell
python scripts/validate-reference-proposals.py
```

## 결과는 무엇인가

이 디렉터리를 쓰면 외부 후보가 실제 변경으로 승격되기 전에 사용자가 범위와 위험을 검토할 수 있습니다. 또한 채택하지 않은 이유도 제안 단위로 남길 수 있습니다.

## 기능 추가/수정 판단 기준

제안서가 반복적으로 같은 필드를 요구하면 템플릿을 추가합니다. 제안서가 실제 변경 내용과 자주 어긋나면 적용 후 검증 항목을 강화합니다.

## 구현 연결 정보

- 제안 영역 정책: `runtime/proposals/README.md`
- 제안 템플릿: `runtime/proposals/reference-adoption/_template.md`
- 제안 검증 스크립트: `scripts/validate-reference-proposals.py`
- 제안 초안 생성 스크립트: `scripts/create-reference-proposal.py`
- 외부 사례 탐색 워크플로: `docs/REFERENCE_DISCOVERY_WORKFLOW.md`
- 후보 카드: `research/reference-candidates/`
- 활동 로그: `runtime/activity-log.jsonl`
