# 예약 워크플로

## 한 문장 정의

이 영역은 반복 실행할 워크플로의 트리거, 주기, 담당 에이전트, 산출물, 승인 조건을 문서화하는 곳입니다.

## 무엇을 하는가

현재 시스템은 예약 작업을 자동 실행하지 않습니다. 대신 나중에 자동화를 연결할 수 있도록 반복 워크플로의 계약을 문서로 남깁니다.

## 왜 필요한가

반복 작업을 말로만 정하면 언제, 누가, 어떤 입력으로, 무엇을 만들고, 어디까지 승인 없이 할 수 있는지 흐려집니다. 예약 워크플로 문서는 반복 실행을 안전하게 만들기 위한 사전 계약입니다.

## 일정 템플릿

```markdown
## <schedule-name>

- Workflow:
- Trigger:
- Frequency:
- Owner agent:
- Heartbeat:
- Timeout:
- Resume from:
- Input sources:
- Output artifacts:
- Approval required:
- Last run:
- Next run:
```

## 권장 예약 후보

- 일일 프로젝트 상태 요약: 활동 로그와 막힘을 요약합니다.
- 주간 검증 리뷰: 검증 결과와 피벗 신호를 비교합니다.
- 구현 후 검증: 구현 역할이 완료되면 독립 검증 역할이 결과를 확인합니다.

## 어떻게 동작하는가

예약 워크플로도 일반 에이전트 작업과 같은 권한 기준을 따릅니다. 산출물은 기본적으로 제안이나 검증 결과로 남기고, 반복 실행은 에이전트 실행 로그에 기록합니다. 모든 반복 실행에는 목표 계보가 필요합니다.

## 결과는 무엇인가

예약 워크플로를 문서화하면 반복 작업의 소유자, 입력, 출력, 승인 조건이 명확해집니다. 나중에 자동화를 붙여도 사람이 검토해야 하는 지점이 유지됩니다.

## 기능 추가/수정 판단 기준

새 예약 작업은 반복 가치가 있고, 입력과 출력이 명확하고, 권한과 중단 조건이 정해졌을 때만 추가합니다. 한 번만 실행할 작업은 예약 워크플로로 만들지 않습니다.
