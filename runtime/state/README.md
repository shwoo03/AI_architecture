# 런타임 상태

## 한 문장 정의

런타임 상태 영역은 훅이나 자동화 단계가 서로 직접 호출하지 않고도 임시 상태를 공유하게 하는 곳입니다.

## 무엇을 하는가

상태 파일은 자동화가 다음 단계에 전달할 임시 정보를 담습니다. 장기 결정이나 감사 증거는 상태 파일에 두지 않고 활동 로그에 남깁니다.

## 왜 필요한가

훅 스타일 자동화는 관찰, 상태 저장, 후속 처리, 로그 기록이 분리될 수 있습니다. 상태 파일은 이 사이를 연결하지만, 프로젝트의 진짜 기록이 되면 안 됩니다.

## 어떻게 동작하는가

기본 흐름은 다음과 같습니다.

```text
이벤트 관찰 -> 임시 상태 기록 -> 다음 단계가 상태 읽기 -> 결정은 활동 로그에 기록
```

## 규칙

- 상태 파일은 기계가 읽기 쉬운 JSON을 사용합니다.
- 상태는 임시 정보입니다.
- 비밀 정보는 저장하지 않습니다.
- 프로젝트 결정의 source of truth로 쓰지 않습니다.
- 상태가 의사결정에 영향을 주면 그 결정은 활동 로그에 남깁니다.

## 결과는 무엇인가

런타임 상태를 분리하면 자동화 단계끼리 필요한 신호를 주고받을 수 있습니다. 동시에 장기 기록은 로그에 남아 감사 가능성이 유지됩니다.

## 기능 추가/수정 판단 기준

새 상태 파일은 후속 자동화가 실제로 읽을 때만 추가합니다. 사람이 읽어야 하는 결정이나 검증 결과는 상태 파일이 아니라 로그, 검증 결과, 제안으로 남깁니다.
